# Lann 后台管理系统需求文档

## 1. 概述

### 1.1 文档目的
本文档定义 Lann 泰国贷款应用后台管理系统的功能需求，为开发团队提供清晰的功能规范和业务流程指导。

### 1.2 系统定位
后台管理系统是 Lann 平台的核心运营工具，服务于：
- **运营人员**：日常业务审核、用户管理、产品配置
- **风控人员**：信用审核、逾期管理、催收任务
- **管理人员**：数据看板、业务分析、决策支持
- **系统管理员**：权限管理、系统配置、审计日志

### 1.3 设计原则
- **安全性**：严格的权限控制、操作审计、数据加密
- **易用性**：清晰的导航、直观的操作、高效的审核流程
- **可配置**：产品、费率、流程等支持动态配置
- **可扩展**：模块化设计，支持业务快速增长

## 2. 核心功能模块

### 2.1 用户管理

#### 2.1.1 用户列表
**功能描述**：展示所有注册用户的基本信息，支持搜索和筛选

**列表字段**：
| 字段 | 类型 | 说明 |
|------|------|------|
| 用户 ID | String | 唯一标识 |
| 姓名 | String | 用户真实姓名 |
| 手机号 | String | 登录账号 |
| 注册时间 | DateTime | 用户注册日期 |
| KYC 状态 | Enum | 待审核/已通过/已拒绝 |
| 信用评分 | Number | 当前信用分数 (300-850) |
| 授信额度 | Number | 可用额度 (泰铢) |
| 账户状态 | Enum | 激活/冻结/注销 |
| 最后登录 | DateTime | 最后登录时间 |

**筛选条件**：
- KYC 状态（全部/待审核/已通过/已拒绝）
- 账户状态（全部/激活/冻结/注销）
- 注册时间范围
- 信用评分范围
- 手机号/姓名模糊搜索

**操作**：
- 查看详情
- KYC 审核（针对待审核用户）
- 冻结/解冻账户
- 注销账户

#### 2.1.2 用户详情
**功能描述**：展示用户完整信息，包括基本信息、借款记录、还款记录等

**信息模块**：
1. **基本信息**
   - 个人信息（姓名、身份证号、出生日期）
   - 联系方式（手机号、邮箱、地址）
   - 工作信息（雇主、职位、月收入）

2. **KYC 信息**
   - 身份证正反面照片
   - 人脸识别照片
   - 审核记录（审核人、时间、意见）

3. **信用信息**
   - 信用评分及历史变化
   - 授信额度及使用率
   - 黑名单状态

4. **借款记录**
   - 历史借款列表（借款金额、期限、状态）
   - 当前在借订单

5. **还款记录**
   - 历史还款记录
   - 逾期记录

6. **操作日志**
   - 账户操作历史（冻结、解冻、额度调整等）

#### 2.1.3 KYC 审核流程
**功能描述**：处理用户提交的 KYC 认证申请

**审核流程**：
```
用户提交 KYC → 自动初审 → 人工复审 → 审核结果通知
```

**审核步骤**：
1. **自动初审**（系统自动）
   - 身份证 OCR 识别验证
   - 人脸比对（与身份证照片）
   - 黑名单检查
   - 年龄验证（18-60 岁）

2. **人工复审**（运营人员）
   - 查看身份证照片清晰度
   - 确认人脸一致性
   - 核实工作信息
   - 填写审核意见

**审核结果**：
- **通过**：用户 KYC 状态更新为"已通过"，进入信用评估流程
- **拒绝**：用户 KYC 状态更新为"已拒绝"，发送拒绝原因通知
- **退回补充**：用户需重新提交资料

**审核时效**：
- 自动初审：实时（< 1 分钟）
- 人工复审：24 小时内

#### 2.1.4 用户状态管理
**功能描述**：管理用户账户生命周期状态

**状态定义**：
| 状态 | 说明 | 可操作 |
|------|------|--------|
| 激活 | 正常使用 | 冻结、注销 |
| 冻结 | 暂停使用（逾期/风控） | 解冻、注销 |
| 注销 | 永久禁用 | 无 |

**状态变更规则**：
- **激活 → 冻结**：需填写冻结原因（逾期超过 X 天/风控触发/用户申请）
- **冻结 → 激活**：需满足解冻条件（还清逾期/风控解除）
- **任意状态 → 注销**：需确认无在借订单或已结清

### 2.2 信用管理

#### 2.2.1 信用评分复审
**功能描述**：对用户信用评分进行人工复审和调整

**触发场景**：
- 用户对信用评分有异议申诉
- 系统检测到异常评分变化
- 定期复审（高风险用户）

**复审信息**：
- 当前信用评分及评分模型版本
- 评分因子明细（收入、负债、历史履约等）
- 用户申诉材料（如有）
- 历史评分变化曲线

**复审操作**：
- 维持原评分
- 手动调整评分（需填写理由）
- 重新计算评分（触发模型重算）

#### 2.2.2 额度调整审批
**功能描述**：管理用户授信额度的调整流程

**调整类型**：
| 类型 | 触发条件 | 审批流程 |
|------|----------|----------|
| 提额 | 良好还款记录/主动申请 | 系统评估 → 人工确认 |
| 降额 | 风险上升/逾期记录 | 系统自动 → 通知用户 |
| 冻结额度 | 严重逾期/风控触发 | 风控人员手动 |

**提额规则**：
- 连续 3 期按时还款：可提额 10-20%
- 收入证明更新：根据新收入重新评估
- 最高额度上限：50,000 泰铢

**审批流程**：
```
提额申请 → 系统评估 → 风控审核 → 审批通过 → 额度更新 → 通知用户
```

#### 2.2.3 黑名单管理
**功能描述**：管理信用黑名单用户

**黑名单类型**：
| 类型 | 触发条件 | 期限 |
|------|----------|------|
| 严重逾期 | 逾期超过 90 天 | 永久 |
| 欺诈行为 | 提供虚假资料 | 永久 |
| 恶意逃废债 | 失联/拒绝还款 | 5 年 |
| 高风险用户 | 多次逾期/多头借贷 | 1 年 |

**黑名单操作**：
- 添加黑名单（手动/自动）
- 移除黑名单（需审批）
- 黑名单查询
- 黑名单导出

**黑名单影响**：
- 无法再次借款
- KYC 审核自动拒绝
- 同步至行业共享黑名单

### 2.3 借款管理

#### 2.3.1 借款申请审核
**功能描述**：处理用户借款申请，支持自动审核和人工审核

**审核模式**：
| 模式 | 适用场景 | 审核流程 |
|------|----------|----------|
| 自动审核 | 小额/优质客户 | 系统规则 → 自动放款 |
| 人工审核 | 大额/首次借款/高风险 | 系统初审 → 人工复审 |

**自动审核规则**：
- 借款金额 ≤ 5,000 泰铢
- 信用评分 ≥ 650
- 历史借款履约率 100%
- 非黑名单用户

**人工审核要点**：
- 借款用途合理性
- 还款能力评估（收入/负债比）
- 历史借款记录
- 近期信用变化

**审核结果**：
- **通过**：进入合同签署流程
- **拒绝**：发送拒绝通知（附原因）
- **补充材料**：通知用户补充资料

**审核时效**：
- 自动审核：实时（< 1 分钟）
- 人工审核：2 小时内

#### 2.3.2 合同管理
**功能描述**：管理借款合同的生成、签署和归档

**合同类型**：
- 电子借款合同
- 抵押合同（如有）
- 担保合同（如有）

**合同状态**：
| 状态 | 说明 |
|------|------|
| 待签署 | 合同已生成，等待用户签署 |
| 已签署 | 用户已完成签署 |
| 已归档 | 合同已归档存储 |
| 已终止 | 合同提前终止 |

**合同操作**：
- 查看合同详情
- 下载合同 PDF
- 重新发送签署链接
- 合同终止（特殊情况）

#### 2.3.3 放款审批
**功能描述**：审核并执行放款操作

**放款流程**：
```
合同签署完成 → 放款审核 → 财务打款 → 放款成功 → 通知用户
```

**审核要点**：
- 合同签署完整性
- 收款账户信息准确性
- 风控复核（放款前最后检查）

**放款方式**：
- 银行转账（主要方式）
- 第三方支付（PromptPay）

**放款时效**：
- 审核通过后 1 小时内放款
- 银行到账时间：工作日 1-2 小时

### 2.4 还款管理

#### 2.4.1 还款计划调整
**功能描述**：根据特殊情况调整用户还款计划

**调整类型**：
| 类型 | 适用场景 | 审批级别 |
|------|----------|----------|
| 延期还款 | 临时困难/特殊情况 | 运营主管 |
| 分期调整 | 收入变化/负债变化 | 风控主管 |
| 减免罚息 | 系统错误/特殊情况 | 财务主管 |

**调整规则**：
- 延期还款：最多延期 30 天，仅限 1 次
- 分期调整：重新计算分期计划，总期限不变
- 减免罚息：需提供合理理由和证明

**审批流程**：
```
用户申请 → 客服初审 → 主管审批 → 系统调整 → 通知用户
```

#### 2.4.2 逾期管理
**功能描述**：管理逾期订单和逾期用户

**逾期等级**：
| 等级 | 逾期天数 | 处理方式 |
|------|----------|----------|
| M0 | 1-7 天 | 短信/APP 提醒 |
| M1 | 8-30 天 | 电话催收 |
| M2 | 31-60 天 | 上门催收/法务介入 |
| M3 | 61-90 天 | 委外催收 |
| M4 | 90+ 天 | 坏账核销/法律诉讼 |

**逾期信息**：
- 逾期订单列表（逾期金额、天数、罚息）
- 逾期用户统计
- 催收记录

**逾期操作**：
- 查看逾期详情
- 记录催收沟通
- 分配催收任务
- 标记还款承诺

#### 2.4.3 催收任务分配
**功能描述**：将逾期订单分配给催收人员处理

**分配规则**：
- 按逾期等级自动分配
- 按催收员工作量均衡分配
- 支持手动调整

**催收任务信息**：
| 字段 | 说明 |
|------|------|
| 任务 ID | 唯一标识 |
| 用户信息 | 姓名、手机号、欠款金额 |
| 逾期等级 | M0-M4 |
| 分配时间 | 任务创建时间 |
| 催收员 | 负责人 |
| 任务状态 | 待处理/处理中/已完成 |
| 最后联系 | 最后沟通时间 |

**催收记录**：
- 联系时间
- 联系方式（电话/短信/上门）
- 沟通内容
- 用户承诺
- 下次跟进时间

### 2.5 产品配置

#### 2.5.1 借款产品 CRUD
**功能描述**：创建、编辑、删除借款产品

**产品信息**：
| 字段 | 类型 | 说明 |
|------|------|------|
| 产品 ID | String | 唯一标识 |
| 产品名称 | String | 如"现金贷"、"消费贷" |
| 产品描述 | Text | 产品介绍 |
| 最小金额 | Number | 最低借款金额 (泰铢) |
| 最大金额 | Number | 最高借款金额 (泰铢) |
| 最长期限 | Number | 最长借款期限 (天) |
| 最短期限 | Number | 最短借款期限 (天) |
| 利率类型 | Enum | 固定/浮动 |
| 基础利率 | Number | 日利率/月利率 |
| 服务费 | Number | 服务费比例 |
| 状态 | Enum | 上架/下架 |

**产品操作**：
- 创建新产品
- 编辑产品信息
- 上架/下架产品
- 查看产品详情
- 删除产品（无在借订单）

#### 2.5.2 费率配置
**功能描述**：配置借款产品的费率和费用

**费率类型**：
| 类型 | 说明 | 计算方式 |
|------|------|----------|
| 利息 | 资金使用成本 | 本金 × 利率 × 期限 |
| 服务费 | 平台服务费 | 本金 × 服务费比例 |
| 管理费 | 账户管理费 | 固定金额/比例 |
| 逾期罚息 | 逾期惩罚 | 逾期金额 × 罚息率 × 逾期天数 |
| 提前还款手续费 | 提前结清费用 | 剩余本金 × 手续费比例 |

**费率规则**：
- 支持按产品配置不同费率
- 支持按用户等级差异化费率
- 支持促销活动临时费率
- 费率变更需审批

#### 2.5.3 期限配置
**功能描述**：配置借款产品的期限选项

**期限配置**：
- 可选期限列表（如 7 天、14 天、30 天、60 天、90 天）
- 默认期限（用户首次选择）
- 期限与利率关联（不同期限不同利率）

**期限规则**：
- 支持灵活组合（如 7-30 天任意天数）
- 支持固定期限选项（如只能选 7/14/30 天）
- 首期产品默认 30 天

### 2.6 渠道管理

#### 2.6.1 还款渠道配置
**功能描述**：配置用户可用的还款渠道

**渠道类型**：
| 渠道 | 类型 | 说明 |
|------|------|------|
| 银行转账 | 线下 | 用户转账至指定账户 |
| PromptPay | 第三方支付 | 泰国本地支付 |
| TrueMoney | 电子钱包 | 泰国流行钱包 |
| 7-11 便利店 | 线下 | 柜台现金还款 |
| 自动扣款 | 代扣 | 绑定银行卡自动扣款 |

**渠道配置**：
- 渠道名称
- 渠道类型
- 渠道账号信息
- 渠道状态（启用/禁用）
- 手续费配置

#### 2.6.2 渠道状态监控
**功能描述**：监控各还款渠道的可用性和交易状态

**监控指标**：
| 指标 | 说明 | 告警阈值 |
|------|------|----------|
| 渠道可用性 | 渠道是否正常 | 不可用 |
| 交易成功率 | 成功交易占比 | < 95% |
| 平均处理时间 | 交易处理时长 | > 30 秒 |
| 日交易量 | 每日交易笔数 | 异常波动 |

**告警方式**：
- 系统通知（后台消息）
- 短信告警（负责人）
- 邮件告警（技术团队）

#### 2.6.3 渠道费率配置
**功能描述**：配置各还款渠道的手续费率

**费率配置**：
| 渠道 | 费率类型 | 费率 | 承担方 |
|------|----------|------|--------|
| 银行转账 | 固定 | 0 泰铢 | 平台 |
| PromptPay | 比例 | 0.5% | 用户 |
| TrueMoney | 比例 | 1% | 用户 |
| 7-11 便利店 | 固定 | 20 泰铢 | 用户 |
| 自动扣款 | 固定 | 0 泰铢 | 平台 |

**费率规则**：
- 支持按渠道配置不同费率
- 支持按金额阶梯费率
- 支持促销活动免手续费

### 2.7 报表统计

#### 2.7.1 业务数据看板
**功能描述**：展示核心业务指标的实时数据

**核心指标**：
| 指标 | 说明 | 更新频率 |
|------|------|----------|
| 累计注册用户 | 平台总用户数 | 实时 |
| 今日新增用户 | 当日新增用户 | 实时 |
| 累计放款金额 | 平台总放款金额 | 实时 |
| 今日放款金额 | 当日放款金额 | 实时 |
| 在借订单数 | 当前未结清订单 | 实时 |
| 逾期订单数 | 当前逾期订单 | 实时 |
| 累计收入 | 平台总收入 | 实时 |
| 今日收入 | 当日收入 | 实时 |

**趋势图表**：
- 用户增长趋势（近 30 天）
- 放款金额趋势（近 30 天）
- 收入趋势（近 30 天）
- 逾期率趋势（近 30 天）

#### 2.7.2 逾期率统计
**功能描述**：统计和分析逾期数据

**统计维度**：
- 按逾期等级统计（M0-M4）
- 按产品类型统计
- 按用户等级统计
- 按放款月份统计（账龄分析）

**逾期率指标**：
| 指标 | 计算公式 |
|------|----------|
| 当前逾期率 | 逾期订单数 / 在借订单数 |
| M1+ 逾期率 | M1 及以上订单数 / 在借订单数 |
| M3+ 逾期率 | M3 及以上订单数 / 在借订单数 |
| 坏账率 | 核销金额 / 累计放款金额 |

#### 2.7.3 收入统计
**功能描述**：统计平台收入情况

**收入构成**：
- 利息收入
- 服务费收入
- 罚息收入
- 其他收入

**统计维度**：
- 按日/周/月统计
- 按产品类型统计
- 按收入类型统计

#### 2.7.4 用户增长统计
**功能描述**：统计用户增长和转化情况

**增长指标**：
| 指标 | 说明 |
|------|------|
| 新增用户数 | 当日/周/月新增 |
| 用户增长率 | 环比/同比增长 |
| KYC 转化率 | KYC 通过数 / 注册用户数 |
| 授信转化率 | 获得额度数 / KYC 通过数 |
| 借款转化率 | 借款用户数 / 授信用户数 |

**转化漏斗**：
```
注册用户 → KYC 认证 → 获得授信 → 首次借款 → 复借
```

## 3. 非功能需求

### 3.1 性能要求
- 页面加载时间：< 3 秒
- 接口响应时间：< 500ms
- 支持并发用户：≥ 100
- 数据查询响应：< 2 秒（百万级数据）

### 3.2 安全要求
- 登录验证：账号密码 + 短信验证码（可选）
- 权限控制：基于角色的访问控制（RBAC）
- 操作审计：所有操作记录日志
- 数据加密：敏感数据加密存储
- 会话管理：超时自动登出（30 分钟）

### 3.3 可用性要求
- 系统可用性：≥ 99.5%
- 数据备份：每日自动备份
- 故障恢复：RTO < 4 小时，RPO < 1 小时

### 3.4 兼容性要求
- 浏览器：Chrome 80+、Safari 13+、Edge 80+
- 分辨率：支持 1366×768 及以上
- 移动端：支持平板设备访问

## 4. 业务流程

### 4.1 用户 KYC 审核流程
```
用户提交 KYC → 系统自动初审 → [通过] → 人工复审 → [通过] → KYC 完成
                                    ↓                    ↓
                                [不通过]              [不通过]
                                    ↓                    ↓
                              通知用户补充           通知用户拒绝
```

### 4.2 借款审核流程
```
用户提交借款 → 系统自动评估 → [自动通过] → 生成合同 → 用户签署 → 放款
                                  ↓
                            [需要人工]
                                  ↓
                            人工审核 → [通过] → 生成合同 → ...
                                           ↓
                                       [拒绝]
                                           ↓
                                      通知用户
```

### 4.3 逾期催收流程
```
订单逾期 → 系统自动分配催收任务 → 催收员联系用户 → 记录催收结果
                                              ↓
                                        [承诺还款]
                                              ↓
                                        跟踪还款情况
                                              ↓
                                        [未履约] → 升级催收等级
```

## 5. 与现有系统融合

### 5.1 与用户端 APP 的关系
- 后台管理用户数据来源于 APP 注册用户
- 后台审核结果实时同步至 APP
- 后台配置的产品信息在 APP 端展示

### 5.2 与后端 API 的关系
- 后台调用后端 API 获取数据
- 后台操作通过后端 API 执行
- 共用同一套数据库和认证体系

### 5.3 与风控系统的关系
- 风控评分结果同步至后台展示
- 后台人工审核可覆盖风控决策
- 黑名单数据双向同步
