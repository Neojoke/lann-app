## Charter Lite v2.0 - Android 发布 + 用户进件流程

**创建日期:** 2026-03-16 13:28  
**状态:** 🟡 待启动  
**优先级:** P0

### 新增目标
1. **Android App 打包发布** - 使用 Capacitor 生成 Android APK
2. **用户进件流程** - 资料填写 → 信用评估 → 授信额度 → 借款

### 用户进件流程设计

**流程步骤:**
```
注册/登录 → 填写个人资料 → 提交审核 → 信用评估 → 授予额度 → 可借款
```

**需要收集的资料:**
1. **基本信息:** 姓名、身份证号、出生日期、性别
2. **联系信息:** 手机号、邮箱、居住地址
3. **工作信息:** 公司名称、职位、月收入、工作地址
4. **紧急联系人:** 姓名、关系、手机号
5. **银行账户:** 银行名称、账号、PromptPay 绑定

**信用评估逻辑:**
- 基于手机号 + 基本信息 + 工作信息
- 首次授信额度：1,000-10,000 THB
- 良好还款后逐步提升额度

### 技术实现方案

**前端页面:**
- `Profile.tsx` - 用户资料页面
- `CreditApply.tsx` - 信用申请页面
- `CreditStatus.tsx` - 审核状态页面

**服务层:**
- `user.service.ts` - 用户资料管理
- `credit.service.ts` - 信用评估和额度管理

**后端 API:**
- `POST /api/user/profile` - 提交用户资料
- `POST /api/credit/apply` - 申请信用额度
- `GET /api/credit/status` - 查询审核状态和额度

### Android 打包方案

**Capacitor 配置:**
- 目标 SDK: Android 13 (API 33)
- 最低 SDK: Android 8 (API 26)
- 应用签名：生成正式签名密钥
- 权限配置：相机、存储、网络等

**发布流程:**
1. `npx cap add android` - 添加 Android 平台
2. `npx cap sync` - 同步代码
3. Android Studio 打开项目
4. 配置签名密钥
5. 生成 APK/AAB
