## 🚫 禁止行为

### 1. 禁止虚报测试结果

**❌ 错误做法:**
- 创建测试文件但未执行，声称"测试通过"
- 配置测试但未运行，报告"覆盖率 90%+"
- 未连接真实设备/模拟器，声称"E2E 测试完成"

**✅ 正确做法:**
- 执行测试后报告真实结果
- 未执行时明确说明"测试文件已创建，待执行"
- 提供测试执行日志和报告

## ✅ 测试验证清单

## 🔧 测试执行命令

### 单元测试

```bash
# 后端单元测试
cd backend
npm test -- --coverage

# 前端单元测试
cd mobile-app
npm test -- --coverage
```

### 集成测试

```bash
# API 集成测试
cd backend
npm run test:integration
```

### E2E 测试

```bash
# 1. 启动模拟器
emulator -avd LannDemo

# 2. 安装 APK
adb install -r mobile-app/android/app/build/outputs/apk/debug/app-debug.apk

# 3. 运行测试
maestro test test/e2e/flows/ --reporter html

# 4. 查看报告
open .maestro/report.html
```

## 📊 测试状态标识

| 标识 | 含义 | 说明 |
|------|------|------|
| ✅ | 已执行并通过 | 测试实际运行且通过 |
| ❌ | 已执行但失败 | 测试实际运行但失败 |
| ⏭️ | 跳过 | 测试配置跳过 |
| 📝 | 文件已创建 | 测试文件存在，未执行 |
| ⏳ | 待执行 | 测试计划中 |

---

## 🎯 覆盖度计算标准

**真实覆盖度 = 实际执行的测试覆盖的代码 / 总代码**

**❌ 错误计算:**
- 测试文件数量 / 源代码文件数量
- 配置的覆盖度目标 / 实际覆盖度

**✅ 正确计算:**
- 使用覆盖率工具 (c8/istanbul)
- 基于实际执行的测试
- 生成 HTML/JSON 报告
